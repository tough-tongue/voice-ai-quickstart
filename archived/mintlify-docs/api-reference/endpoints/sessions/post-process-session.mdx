---
title: "Post-Process Session"
api: "POST /v2/sessions/{session_id}/post-process"
description: "Re-run analysis and/or extraction on an existing session"
---

Trigger post-session processing jobs in the background. Use this to re-run
analysis after updating a scenario's evaluation rubric, or to run variable
extraction on sessions that predate the feature.

Calling this endpoint always **re-runs** the requested jobs — results overwrite
any existing analysis or extraction data.

## Path Parameters

<ParamField path="session_id" type="string" required>
  ID of the session to post-process
</ParamField>

## Request Parameters

<ParamField body="run_analysis" type="boolean" default="false">
  Re-run session analysis: debrief, next steps, scoring, report card
</ParamField>

<ParamField body="run_extraction" type="boolean" default="false">
  Re-run variable extraction. Requires `extraction_vars` to be configured
  on the scenario.
</ParamField>

<RequestExample>
```json
{
  "run_analysis": true,
  "run_extraction": false
}
```
</RequestExample>

<ResponseExample>
```json 200
{
  "status": "processing",
  "session_id": "session_abc123"
}
```
</ResponseExample>

<Note>
Processing runs asynchronously. Poll [Get Session](/api-reference/endpoints/sessions/get-session)
until `status` is `completed` to retrieve the updated results.
</Note>
