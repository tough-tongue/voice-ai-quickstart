---
title: "Create Session"
api: "POST /v2/session"
description: "Create a completed session from a transcript or recording URL"
---

Upload an existing conversation — as raw transcript text or an audio/video recording
URL — and have ToughTongue AI process it as a session. Analysis runs in the background
after creation.

Requires VIEW access to the scenario.

## Request Parameters

<ParamField body="scenario_id" type="string" required>
  The scenario this session belongs to
</ParamField>

<ParamField body="transcript" type="string">
  Plain-text transcript of the conversation. Provide either `transcript` or
  `recording_url` — not both.
</ParamField>

<ParamField body="recording_url" type="string">
  URL to an audio or video recording. Automatically transcribed unless the
  scenario has multimodal analysis enabled, in which case the raw media is used.
</ParamField>

<ParamField body="user_name" type="string">
  Display name for the session participant
</ParamField>

<ParamField body="user_email" type="string">
  Email address for the session participant
</ParamField>

<ParamField body="metadata" type="object">
  Arbitrary custom metadata to attach to the session
</ParamField>

<ParamField body="user_metadata" type="object">
  Filterable key-value metadata (string values only). Queryable via the
  sessions list endpoint.
</ParamField>

<ParamField body="force_analysis" type="boolean" default="false">
  Force analysis to run even if the scenario has analysis disabled
</ParamField>

<RequestExample>
```json
{
  "scenario_id": "693960de75f58a7b74c835bf",
  "recording_url": "https://storage.example.com/recordings/call-jan-15.mp3",
  "user_email": "candidate@example.com",
  "user_name": "Jane Smith"
}
```
</RequestExample>

<ResponseExample>
```json 200
{
  "id": "session_abc123",
  "scenario_id": "693960de75f58a7b74c835bf",
  "status": "processing",
  "created_at": "2026-01-15T10:30:00Z"
}
```
</ResponseExample>
