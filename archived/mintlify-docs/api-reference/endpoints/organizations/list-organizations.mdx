---
title: "List Organizations"
api: "GET /v2/organizations"
description: "List organizations the authenticated user belongs to"
---

Returns all organizations associated with the PAT holder's account.

Each organization's `id` should be passed as the `X-TT-ORG` request header on
subsequent API calls to scope them into that organization's context — enabling
org-wide data access, shared scenarios, and team billing.

<ResponseExample>
```json 200
{
  "organizations": [
    {
      "id": "org_abc123",
      "name": "Acme Corp",
      "role": "admin"
    }
  ]
}
```
</ResponseExample>

## Using the org context

Once you have an `id`, add the header to any API request:

```bash
curl https://api.toughtongueai.com/api/public/sessions \
  -H "Authorization: Bearer $TTAI_PAT_TOKEN" \
  -H "X-TT-ORG: org_abc123"
```

This scopes the response to all sessions across the organization, not just the
authenticated user's own sessions.
