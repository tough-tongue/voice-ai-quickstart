---
title: "Create SAT (Self)"
api: "POST /scenario-access-token/me"
description: "Create a Scenario Access Token where the PAT holder is the session runner"
---

Self-billed SAT — the authenticated API user is recorded as the session participant.
Public scenarios bypass access control; private scenarios require the PAT holder to
be the creator or have READ access.

Unlike the standard SAT endpoint, no `email` override is needed — the token is
automatically scoped to the requesting user's identity.

<RequestExample>
```json
{}
```
</RequestExample>

<Note>
The request body schema mirrors `POST /scenario-access-token`. All the same
optional fields (`scenario_id`, `duration_hours`, `email`) are accepted —
the difference is that `email` defaults to the PAT holder's email.
</Note>
